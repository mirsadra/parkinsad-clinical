When does the tech enter community meds into the app?

This determines the whole workflow. Which fits how technicians actually work?

A

Before bedside — from screen

Tech opens SystmOne/HIE on one screen, enters meds into our app on another. App shows discrepancies vs inpatient FHIR meds immediately. Tech prints our app's comparison view and takes it to the patient — one structured sheet instead of two printouts.

SCR/HIE (screen) → enter into app → comparison view → print → bedside → annotate → return → generate Task note
B

After bedside — from annotated printout

Tech still prints from SystmOne/HIE as today and goes bedside as normal. On return, they open our app and enter what they found, using their annotated printout as the source. App then compares, highlights discrepancies, and generates the Task note.

Print (as today) → bedside → annotate → return → enter into app → comparison + Task note generated
C

Flexible — app supports both moments

Tech can use the app either way. If they enter before bedside they get a better printout. If they enter after, they still get the Task note generated. The app doesn't force a specific moment.

Either flow works — same app, same comparison, same Task note output

My lean: B first, C as the goal. Starting with after-bedside entry requires the least behaviour change and still delivers the biggest win (Task note + discrepancy summary). Once adopted, offering the before-bedside flow is a natural upgrade.
